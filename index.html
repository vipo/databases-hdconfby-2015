Cassandra

Real Cassandra ring

Что хорошо

  • Может хранить тьму данных
  • Масштабируется
  • CQL / составные ключи
  • Она понятна, много книг
  • Достаточно стабильна (падает раз в год)
  • Работает на любом железе

Что плохо

  • Java GC
  • Легко перегреть один элемент в кольце

Hot (and big) spot

Лечение / Калечение

CREATE TABLE t (a int, b int, c int, d int, PRIMARY KEY(a, b, c));
меняем на
CREATE TABLE t (a int, b int, c int, d int, PRIMARY KEY((a, b), c);

A что совсем плохо

Время отзыва multi-get / работа со множествами
  • SELECT * WHERE k in (1, ... n)
  • SELECT * WHERE k = 1; ...; SELECT * WHERE k = n

Redis

... потому что могли

Чего обещают

  • Аварийное переключение
  • Сохранение на диск
  • Супер быстрый

Что есть на самом деле

  • Аварийное переключение работает, если настроить
  • Сохранение на диск ничего не портит, если без перфекционизма
  • Супер быстрый, если не решать проблемы "в лоб"

Redis Sentinel

Persistence

  • AOF — пишем лог, иногда сжимаем
  • RDB — просто снимок данных

Всё бы ничего

Fsync() also blocks the process for all the time needed to complete the write, and if this is not enough, on Linux it will also block all the other threads that are writing against the same file.

Что пробовали

  • Настройка fsync параметров
  • Читать со slave'a
Но ничего и не могло помочь: мы пишем больше чем читаем
